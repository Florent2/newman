Newman::Settings provides the base functionality that is used by Newman’s configuration files. It is currently a thin wrapper on top of Ruby’s OpenStruct construct, but will later add some domain specific validations and transformations for the various configuration options it supports.

Use from_file to create a new Newman::Settings object and populate it with the data contained in the settings file, i.e.

settings = Newman::Settings.from_file(‘config/environment.rb’)

This method is purely syntactic sugar, and is functionally equivalent to the following code:

settings = Newman::Settings.new
settings.load_config(‘config/environment.rb’)

    def self.from_file(filename)
      new.tap { |o| o.load_config(filename) }
    end
¶

A Newman::Settings object is a blank slate upon creation. It simply assigns an empty OpenStruct object for each type of settings data it supports. The imap and smtp fields are used by Newman::Mailer, the service field is used throughout Newman, particular in Newman::Server, and Newman::Application is reserved for application-specific configurations.

In most situations, you will not instantiate a Newman::Settings object directly but instead will make use of a configuration file and the Newman::Settings.from_file method. Newman provides sample configuration files in its examples/ and test/ directories, and applications should do the same. This will help minimize the amount the user needs to discover for themselves what fields can be set.

 
    def initialize
      self.imap        = OpenStruct.new
      self.smtp        = OpenStruct.new
      self.service     = OpenStruct.new
      self.application = OpenStruct.new
    end

    def load_config(filename)
      eval(File.read(filename), binding)
    end

    attr_accessor :imap, :smtp, :service, :application
  end
